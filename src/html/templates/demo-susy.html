Why should we start using Susy instead of Bootstrap?

When Bootstrap and Foundation become the next popular thing in frontend development, they brought something new to the table, the ability for the html (not the just stylesheet) itself to setup the grid layout through different breakpoints possible using pre-made classes, I thought it was great at that, but that's not only where these frontend frameworks were good at, they can also provide your barebone site project an instant cosmetic upgrade using again some premade classes to apply themed styles to your element, not only that, it gives you access to quick javascript goodness in providing features like slideshows, carousels and modal popups to your site.

However, whenever I get a psd to html task and look at the design in psd file, it barely resembles any style themes available to Bootstrap, the designer may either be not aware of Bootstrap styles or really doing custom design for the sake of the brand that the design should be carrying. In the end, I will just end up doing overrides to existing bootstrap styles and just add new styles based on the design (which what I should be doing in the first place). Unless you're not using less or sass (css preprocessors so you can only pickup which bootstrap feature you want to use), you will have your site getting lots of unused styles adding up to your overhead. Fortunately, The Bootstrap website allows you to have a custom build of the Boostrap package you want to use for your site, for example you can only pick the 'grid system' to be in your Bootstrap framework package.

But then, going back to the HTML + CSS coupling style of Bootstrap where you could set the grid system by adding classes to your html element tags, wouldn't it be better to just have these kinds of grid system be handled by the stylesheet files and leave the html alone? Yes, I did said that I thought it was great thing to have, but the number of these classes can confuse the backend coders.
